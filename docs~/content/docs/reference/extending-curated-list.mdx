---
title: "큐레이션 목록 확장"
---

Featured 탭에 패키지를 추가하거나 수정하려면 `Editor/Services/CuratedPackages.cs`를 편집하세요.

## 데이터 모델

```csharp
internal class CuratedPackage
{
    public string Id;          // manifest dependency 키 (예: com.cysharp.unitask)
    public string DisplayName; // 카드에 크게 표시되는 이름
    public string Description; // 한 줄 설명
    public string GitUrl;      // manifest dependency 값 — Git URL 또는 버전 문자열
    public string Repository;  // "GitHub" 버튼이 여는 URL
    public string Category;    // 카드 그루핑 키
}
```

## 항목 추가 예시

```csharp
new CuratedPackage
{
    Id = "com.example.awesome",
    DisplayName = "Awesome Library",
    Description = "Does awesome things for Unity.",
    GitUrl = "https://github.com/example/awesome.git?path=Packages/com.example.awesome",
    Repository = "https://github.com/example/awesome",
    Category = "Tools",
},
```

`CuratedPackages.All` 리스트의 끝(또는 적절한 카테고리 옆)에 위 블록을 추가하면 즉시 Featured 탭에 반영됩니다.

## 권장 사항

* **`Id`는 패키지의 실제 `package.json` 이름과 일치**시키세요. 그래야 Unity가 같은 패키지의 중복 추가/이미 설치 상태를 정확히 인식합니다.
* `GitUrl`에 `?path=...`로 하위 경로를 지정해야 하는 패키지가 많습니다(레포 루트가 Unity 패키지가 아닌 경우). 해당 프로젝트의 README를 참고하세요.
* 버전 고정이 필요한 패키지는 `?path=...#v1.2.3` 형식으로 태그를 붙입니다.
* OpenUPM에서 정식 호스팅되는 패키지면 `GitUrl`에 **버전 문자열**(예: `"3.2.1"`)을 넣어도 됩니다. 단, 그렇게 하려면 사용자가 미리 OpenUPM 스코프 레지스트리를 추가해야 하므로 Git URL 형태가 더 자급자족적입니다.
* `Category`는 자유 문자열이지만, 동일 명칭을 일관되게 쓰면 그루핑이 깔끔합니다.

## 표시 순서

* 카테고리는 알파벳순으로 그룹화됩니다.
* 카테고리 내 카드는 `DisplayName` 알파벳순.

특정 순서를 원한다면 카테고리 명에 prefix(`01 - Async` 같은)를 붙이는 방법이 있지만, 권장하지는 않습니다.
